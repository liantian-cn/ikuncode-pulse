# Project Rules

1. Shell：在 Windows 上默认使用 PowerShell。
2. Python：使用 `uv` 作为包管理器，运行 Python 命令时优先用 `uv run`。
3. Lua：使用 `luacheck` 做检查。
4. 开始任何编码任务前，如果 Git 工作区有未提交内容，先创建一个备份提交。
5. 用户是多年个人业余开发者，必要时请用更通俗的话说明。
6. 以后每次开始写代码前，先把本次实施计划写入 `plan/` 目录下的一个 Markdown 文件。
7. `plan/` 里的计划文件名必须使用 `4位数字序号_标题.md` 格式，例如 `0001_example.md`。
